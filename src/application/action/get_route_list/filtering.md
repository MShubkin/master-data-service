# Фильтрация маршрутов.

Фильтрация маршрутов выполняется в ручке `v1/get/route_list/` в соответствии со
стандартным запросом ФЕ.

Основную трудность представляет фильтрация по критериям. Фильтр расчитан на
значения, а в нашем случае это критерии (или предикаты), и не вполне очевидно,
как определять, попадает ли данный предикат под заданный фильтр. Однако,
считается, что фильтрация маршрутов может быть использована для проверки, или
"отладки", критериев маршрутов -- в фильтры задаются параметры ППЗ/ДС, и
проверяется, попал ли нужный маршрут в выборку. Если попал, то значит, значения
удовлетворяют маршруту, если нет -- значит, нет.

## Фильтры критериев.

Для фильтрации критериев используются только фильтры вида `Equals` и `In`.

Для фильтра `Eq value` по полю `field` возвращаются маршруты, для которых
подходит значение поля `field = value`.

Для фильтра `In value_1, .. value_n` по полю `field` возвращаются маршруты, для
которых подходит хотя бы одно из значений поля `field = value_i`.

При таком подходе фильтрация маршрутов может быть реализована с использованием
поиска маршрутов по набору значений. Из входных фильтров собираются значения по
каждому полю, и для каждого поля, которое используется в фильтре, создается
аргумент `CritArg::AnyAgg`, который подразумевает, что под критерий для
соответствующего поля подходит хотя бы одно значение.
